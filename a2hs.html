How the feature works & what we can't see

  1. Mechanics: Android — browser fires beforeinstallprompt; our banner appears after 60 s of watching (30 s if the user previously dismissed the fullscreen prompt); suppressed if already standalone, dismissed <14 days ago, or the guide/fullscreen prompt is open. Tapping Install opens the native prompt. iOS — banner shows manual Add-to-Home-Screen instructions; no install event is possible.
  2. Accept vs decline is not persisted: the client sends the native-prompt outcome but the server drops it (meta is NULL on all 168 events). The 150 are prompt-answerers, not confirmed installs. One-line server fix, deferred with the dashboard repair.
  3. OS split is not persisted either — the Android vs iOS funnel can't be separated.
  4. No appinstalled listener: browser-menu and iOS installs are invisible except via standalone launches.
  5. Launches from the installed app ARE detectable (launch_mode='standalone'), reliable since Jun 27 — this is the ground-truth install metric.
